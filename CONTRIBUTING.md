Contributing Guide

Thank you for considering a contribution. This repository aims to provide practical, lightweight utilities and documents for AI development and operations.

How to Contribute
- Discuss: Open an issue to propose ideas or report bugs.
- Fork and branch: Create a feature branch for your changes.
- Style: Keep scripts portable and dependency-light. Prefer bash and Python with clear flags and help text.
- Tests: Where feasible, include a simple usage example or smoke test.
- Docs: Update `README.md` and `docs/` where relevant. Describe assumptions and side effects.

Pull Requests
- Keep PRs small and focused.
- Reference related issues.
- Include a short rationale and usage notes.

Security and Secrets
- Do not commit secrets. Use environment variables and secret managers.
- Redact or anonymize sensitive data in examples.
